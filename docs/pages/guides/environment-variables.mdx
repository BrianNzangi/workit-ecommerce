# Environment Variables

## Backend (`backend/.env`)

| Variable | Required | Description |
|----------|----------|-------------|
| `DATABASE_URL` | Yes | PostgreSQL connection string |
| `BETTER_AUTH_SECRET` | Yes | Secret for admin auth sessions |
| `BETTER_AUTH_URL` | Yes | Backend base URL such as `http://localhost:3001` |
| `BACKEND_URL` | Yes | Same as `BETTER_AUTH_URL` |
| `FRONTEND_URL` | Yes | Storefront URL such as `http://localhost:3000` |
| `ADMIN_URL` | Yes | Admin URL such as `http://localhost:3002` |
| `BETTER_AUTH_TRUSTED_ORIGINS` | Yes | Comma-separated allowed origins |
| `CORS_ORIGIN` | Yes | Comma-separated CORS origins |
| `PAYSTACK_SECRET_KEY` | Production | Paystack secret key for payment verification |
| `PORT` | No | Backend port, default `3001` |
| `NODE_ENV` | No | `development` or `production` |
| `DB_AUTO_PUSH` | No | Auto-push schema on startup, default `true` in dev |
| `BETTER_AUTH_COOKIE_PREFIX` | No | Admin cookie prefix, default `admin-auth` |
| `STOREFRONT_AUTH_COOKIE_PREFIX` | No | Storefront cookie prefix, default `store-auth` |
| `BETTER_AUTH_COOKIE_DOMAIN` | No | Cross-subdomain cookie domain such as `.workit.co.ke` |
| `ALGOLIA_APP_ID` | No | Algolia app ID for product search |
| `ALGOLIA_ADMIN_API_KEY` | No | Algolia admin API key |
| `ALGOLIA_PRODUCTS_INDEX` | No | Algolia index name, default `products` |
| `REDIS_URL` | No | Enables Redis-backed cache and rate limiting features |
| `STORAGE_REQUIRED` | No | Fail startup if object storage is unavailable |
| `PRINT_ROUTES` | No | Print all routes on startup |

## Storage (R2 / S3)

| Variable | Description |
|----------|-------------|
| `S3_ENDPOINT` | R2 or S3 endpoint URL |
| `S3_ACCESS_KEY` | Access key ID |
| `S3_SECRET_KEY` | Secret access key |
| `S3_BUCKET` | Bucket name |
| `S3_REGION` | Region, use `auto` for R2 |
| `S3_FORCE_PATH_STYLE` | `false` for R2, `true` only for some local S3-compatible services |
| `S3_AUTO_CREATE_BUCKET` | Usually `false` for R2 and `true` only for local storage you control |
| `PUBLIC_MEDIA_URL` | Public base URL for serving uploaded files |

## Frontend (`frontend/.env.local`)

| Variable | Description |
|----------|-------------|
| `NEXT_PUBLIC_API_URL` | Backend API URL |
| `NEXT_PUBLIC_AUTH_COOKIE_PREFIX` | Must match `STOREFRONT_AUTH_COOKIE_PREFIX` |

## Admin (`admin/.env.local`)

| Variable | Description |
|----------|-------------|
| `NEXT_PUBLIC_API_URL` | Backend API URL |
| `NEXT_PUBLIC_AUTH_COOKIE_PREFIX` | Must match `BETTER_AUTH_COOKIE_PREFIX` |

<Callout type="warning">
  Never commit real `.env` files. Use the example files as templates and keep production secrets in your deployment platform or secret manager.
</Callout>
