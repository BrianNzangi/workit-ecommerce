# Deployment

## Overview

| App | Platform | Notes |
|-----|----------|-------|
| `frontend` | Vercel | Auto-deploy from `main` |
| `admin` | Vercel | Auto-deploy from `main` |
| `backend` | Docker / Railway / Fly.io | Dockerfile included |
| `docs` | Vercel | Auto-deploy from `main` |
| PostgreSQL | Supabase / Railway | Managed PostgreSQL |
| Redis (optional) | Upstash / Railway | Managed cache / rate limiting |
| R2 / S3-compatible storage | Cloudflare R2 / S3 | Object storage |

## Backend Deployment

The active backend (`backend`) uses `Dockerfile.backend` at the repo root.

<Steps>

### Build the Docker image

```bash
docker build -f Dockerfile.backend -t workit-backend .
```

### Set environment variables

Configure all required environment variables in your deployment platform. See [Environment Variables](/guides/environment-variables) for the full list.

### Run the container

```bash
docker run -p 3001:3001 \
  -e DATABASE_URL="postgresql://..." \
  -e BETTER_AUTH_SECRET="..." \
  -e PAYSTACK_SECRET_KEY="..." \
  workit-backend
```

</Steps>

## Frontend And Admin Deployment

<Steps>

### Connect the repo to Vercel

Import the monorepo into Vercel. Set the **Root Directory** to `frontend` or `admin`.

### Configure build settings

| Setting | Value |
|---------|-------|
| Framework | Next.js |
| Build command | `pnpm build` |
| Output directory | `.next` |
| Install command | `pnpm install` |

### Set environment variables

Add all required env vars in the Vercel project settings.

</Steps>

## Docs Deployment

Deploy `docs` the same way on Vercel with the root directory set to `docs`.

## Database Migrations

<Callout type="warning">
  In production, disable `DB_AUTO_PUSH` and run schema changes intentionally before deploying a new backend version.
</Callout>

```bash
pnpm --filter @workit/db generate
DATABASE_URL="postgresql://..." pnpm --filter @workit/db migrate
```

## Health Checks

The backend exposes health check endpoints for load balancers:

```text
GET /health/live   -> 200 if the app process is running
GET /health/ready  -> 200 if the app is ready to serve requests
```

Use `/health/ready` for readiness probes and `/health/live` for liveness probes.
