# Architecture Overview

Workit is a monorepo with three deployable apps sharing common packages.

## System Diagram

```text
Client Layer
  Storefront (Next.js)          Admin Dashboard (Next.js)
          |                              |
          +--------------+---------------+
                         |
                    Backend API
               Fastify 5 + TypeScript
                         |
        +----------------+----------------+
        |                |                |
    PostgreSQL      Redis (optional)   R2 / S3
     Drizzle          Cache / limits    Storage
```

## Key Design Decisions

### Dual Auth System

The platform uses two Better Auth entry paths:

- **Admin auth** at `/auth/*` for admin panel users with role-based permissions
- **Storefront auth** at `/api/auth/*` for customer accounts

### Guest Cart Support

Cart operations support both authenticated customers and guest users via the `x-guest-id` header. On checkout, guest carts can be merged into the customer's account.

### Caching Strategy

Redis is used when `REDIS_URL` is configured:

- Product catalog caching
- Collection and brand caching
- Banner and homepage caching
- Rate limiting counters
